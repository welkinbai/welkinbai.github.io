第一节 使用和避免Null

“I call it my billion-dollar mistake.” - Sir C. A. R. Hoare, on his invention of the null reference

上面是Null引用发明者亲口说的话，大意为：Null是我犯下的一个数百万美元的错误
可见连创始人本人都觉的Null是一个糟糕的发明。
那么Null到底糟糕在哪儿呢？

Null糟糕的地方

如下：

  • Null值的含义不明，比如Map.get(key)返回的Null值，即可以代表在Map中什么也没找到，也可以代表找到的value是Null。在很多程序中，Null可以代表失败，可以代表成功，可以代表任何事。
  • Null值总是伴随着著名的NullPointerException，这个异常之所以著名的原因是因为通常它发生的时候，程序员很难定位错误的根源，只能靠瞎蒙

Null也有一些优点

当Null能被正确使用时，Null是很cheap的（应该就是几乎没有性能和空间开销）。
而且Null在数组对象中是不可避免的（这个我不认同原文观点，可以使用一个专用的引用来专门代表数组中空的元素，而不是通用的Null）

谷歌的建议

谷歌的建议是尽可能地在Null值出现的地方快速失败而不是忍气吞声把Null值接受下来。
我也认为这是个好建议，就地失败可以更容易查出根源问题，比Null值被好几个类传来传去以后才失败要好。

特殊例子的建议

谷歌还提出了以下建议：

  • 如果你打算使用Null放在一个Set里或者作为Map的key，请不要这么做。
  • 如果你打算把Null值作为Map的Value放进去，请考虑分离开来，将Null键值对和非空的键值对分开放，或者维持一个非空键值对（或着空的）key的Set。因为很容易搞混到底是Map里面没有这个value，还是value就是Null。
  • 如果你打算把Null放进List中，如果List不是很大，考虑使用Map<Integer,E>来代替List使用，这样更加高效，也许也更加符合你的应用的需求。
  • 为枚举类型增加一种额外的枚举常量来代表Null值出现的情况，比如java.math.RoundingMode就用UNNECESSARY来代表不需要舍入，并在舍入的时候抛出异常。
  • 如果你真的要使用Null，请使用Collections.unmodifiableList(Lists.newArrayList())来代替ImmutableList，因为后者对Null不友好

谷歌用来替代Null值的兵器——Optional

Guava框架提供了一个叫做Optional<T>的类，它用来包装某个非Null的类型，用来表达存在(present)和不存在(absent)这两个概念。
Optional所包装的对象，要么是存在，要么就是不存在，不会有第三种情况。

Optional提供的方法

首先是用来构造Optional的静态方法：

方法 解释
Optional.of(T) 创造一个包含非空值的Optional对象，如果传入的参数为空，快速失败
Optional.absent() 创造一个不包含值的Optional对象
Optional.fromNullable(T) 创造一个包含非空值的Optional对象，如果传入的参数为空，不会失败，会创建一个不包含值的Optional对象

然后还提供了一些非静态方法，在已经获得的对象上调用：

方法 解释
boolean isPresent() 返回true如果Optional对象中有非空的对象
T get() 返回Optional包装的对象，如果内部没有对象，立即抛出IllegalStateException，显然应该和上面的方法一起使用
T or(T) 返回Optional包装的对象，如果内部没有对象，返回传入的参数
T orNull() 返回Optional包装的对象，如果内部没有对象，返回Null
Set asSet 返回一个不可变的Set，该Set要么包含Optional包装的对象（present情况下），要么就是空的(absent情况下)

Optional的优点

  • 给Null一个可读的名字
  • 自解释的，防止犯错的。它强迫程序员考虑absent时的情况。
  • 当你调用一个方法的时候，你可能会忘记方法的返回值有可能为Null，但是如果方法返回的是一个Optional，当你想要从Optional里面拿出真正的值时，你就不得不考虑Null的情况。Optional的方法会强迫你想起这件事。

其他一些便利的方法

  • MoreObjects.firstNonNull(T,T)这个方法会从左到右返回第一个不是Null的参数，如果两个参数都是Null，就会抛出NullPointerException。当第一个参数是Optional时，它会调用first.or(second)来返回结果（first代表第一个参数，second代表第二个参数）
  • Strings.emptyToNull(String)这个方法返回入参本身，如果入参是个空字符串，那就返回Null。
  • Strings.nullToEmpty(String)这个方法返回入参本身，如果入参是个Null，那就返回空字符串。
  • Strings.isNullOrEmpty(String)这个方法在入参为Null或者空字符串时返回true。

其中，后面三种方法谷歌说主要是为了那些糟糕的API库设计的，有些API库认为空字符串和Null是一个东西，谷歌认为这是不好的，应该把他们的含义分隔开来。

Preconditions提供的方法

提供的方法如下：

方法 描述 不满足条件时抛出的异常
checkArgument(boolean) 检查入参是否为True，用来检查一个方法的入参是否符合要求 IllegalArgumentException
checkNotNull(T) 检查入参是否为Null，为Null抛出异常，不为空则返回入参 NullPointerException
checkElementIndex(int index, int size) 检查index是否为某个数组中合法的index，从0到size(size除外)都是合法的，否则抛出异常 IndexOutOfBoundsException
checkPositionIndex(int index, int size) 检查index是否为某个数组中合法的index，从0到size(包括size)都是合法的，否则抛出异常 IndexOutOfBoundsException
checkPositionIndexes(int start,int end,int size) 检查[start,end)是否是某个数组的一个合法的范围 IndexOutOfBoundsException

一些需要注意的问题

  • 带String参数的自动format过程发生在真的要抛出异常的时候，所以不用担心有性能问题
  • 使用的时候，最好使用静态引用，static import来将该类的调用缩短，缩短后的代码非常简洁，而且由于个别方法直接返回入参，因此可以在一行中嵌套使用
  • 虽然支持在一行中嵌套使用，但是谷歌提倡我们为每个Preconditions检查语句另起一行，这样在debug的时候更加容易和直观，而且如果要传入String参数，放在一行里不太方便。

第三节 链式调用风格比较器

主要是介绍谷歌提供的Ordering这个工具类。它封装了Comparator，将一些静态方法如（Collections.max)封装到内部的实例方法中，并提供链式调用风格来形成复杂的比较器。

由于它实现了Comparator接口，所以可以直接传到Collections.sort方法中使用。

还有其他方式，使用下面这几个静态方法得到一些预置的比较器：

方法 描述
natural() 得到一个自然排序器，对数字和日期都是由小到大排序
allEqual() 得到一个认为所有元素都相等的比较器，即压根不比较结果，不会对元素顺序发生影响(Null值的顺序除外)
arbitrary() 得到一个对象身份比较器，该比较器不比较对象的大小，而是比较它们的系统默认的hashCode（不管你有没有覆盖HashCode方法），a==b发生时，元素的顺序是不确定的，但是在JVM整个生命周期中不再改变（因为它们的hashCode值不会发生变化）
compound(Iterable) 通过一个可迭代的入参得到一个比较器，可迭代入参里存放着数个比较器，这样当第一个比较器比较结果相等时，就用第二个，以此类推
explicit(T t1,T… tn)或者explicit(List) 得到一个部分比较器，该比较器的比较结果和入参的顺序一致，注意，比较的时候传入的比较对象必须都包含在入参里，否则抛出异常
from(comparator) 从一个现成的comparator得到一个Ordering
usingToString() 得到一个字符串比较器，该比较器先针对每个对象应用toString方法，然后根据得到的字符串来比较对象

一些有用的链式调用方法

上面介绍的这些预置的比较器以及我们自己创建的比较器都有一系列额外的方法，来方便地进行链式调用。
需要注意的是，链式调用的时候，要从右往左理解代码，就好像代码是从右往左执行的(compound方法除外)。

方法 描述
reverse() 反转比较结果
nullsFirst() 先把Null值都放在数组最前面，剩下的部分再应用相应规则比较，调用该方法后，我们的比较器就支持含有Null值的数组排序，否则抛出异常
nullsLast() 和上面相反，先把Null值都放在最后面，然后剩余的部分再应用规则比较，同上，该方法使得Ordering比较器容忍Null值在数组中
lexicographical() 该方法一旦调用，比较器的任务就是比较数组了，针对这种情况，先比较数组中的第一个值，在第一个值相同的时候比较第二个值，较短的数组排在前面
compound(Comparator) 这个方法和上面介绍的静态方法功能一致，就是和另一个比较器组合在一起，当上一个比较器比较结果为相等时，应用下一个比较器比较结果，注意该链式调用是唯一一个需要从左往右看的，为了避免搞混，最好在代码链的末尾调用，不要在中间调用
onResultOf(Function) 该方法需要一个入参Function，关于Function的介绍要查看后文了，此处只能说它负责对入参执行一个函数，而本方法使比较器先针对每个比较对象执行该函数，然后根据函数结果来比较大小

链式调用的收尾方法，一些有用的快捷方法

并不是所有方法都是一个链式调用，很多方法将终结链式调用，这些方法往往会产生最终结果：

方法 解释
List greatestOf(Iterable，int k) 返回第一个入参中前k个最大的(排序后靠近末尾)元素
List leastOf(Iterable，int k) 返回第一个入参中前k个最小的(排序后靠近前面)元素
boolean isOrdered(Iterable) 返回入参是否已经是按照当前比较器排好序的了，容忍有相等的元素出现的情况
boolean isStrictlyOrdered(Iterable) 返回入参是否已经是按照当前比较器排好序的了，而且是严格排序，不允许出现相等元素
E Max(Iterable)或者E Max(E a,E b) 入参为元素时，可以有无限多个，为可迭代的对象时，该对象不能为空。返回按照比较器得到的结果中最大的(最靠近末尾)的一个元素
E min(Iterable)或者E Min(E a,E b) 同上，只不过返回的是最小的(最靠近前面)的一个元素
List sortedCopy(Iterable) 返回一个按照比较器排好序的入参的可变副本，而入参的顺序还是原来的顺序不改变。这个方法只有想要不改变入参的时候才调用，它要快于手工地复制一份然后在sort，但是慢于下面这个方法
ImmutableList immutableSortedCopy(Iterable) 同上，唯一的不同是得到的是一个不可变的数组副本，速度快于上面的方法，如果你之后不会再修改结果只是读取的话，推荐用这个方法
