第一期

技术文章

1.基于Redis的分布式锁到底安全吗？

你有没有想过，Redis实现分布式的锁真的安全吗？
对此，Redis的原作者antirez的结论是，只要按照他提出的Redlock规范来实现，就是安全的。然而分布式系统的专家Martin Kleppmann则不赞同，他写了一篇博客揭露Redlock实现的漏洞，就这样，两位大师在著名的论坛Hacker News上开始了激烈的讨论，众多分布式系统专家都参与其中。本文忠实地记录了讨论过程，看完绝对令你受益匪浅。

2.Spring Boot入门

虽然作者语言简略，但是示例代码都贴出来了，全部浏览一遍，不敢保证你能对spring boot深入了解多少，但是至少脑海中构建起了spring boot是什么，能做什么，怎么做的概念。后续还会推出更多spring boot的技术文章哟。

3.一文让你明白Java字节码

我们都知道，JAVA的源文件.java是不能运行的，只有编译成.class的字节码文件才能运行，但是当你用文本编辑器打开.class文件，映入眼帘的一堆16进制编码一定会让你一脸懵逼。
.class文件是怎么构成的？了解它能给我们带来哪些启发？你能不用反编译器，用肉眼识别字节码文件吗？阅读本文兴许能找到答案。

推荐技术（或框架）：

deeplearning4j

一个开源的，基于JAVA的深度学习框架，即使你是一个人工智能小白，利用此框架也能轻松构建一个人工智能神经网络（我没试过，宣传语是这么说的）。
随着谷歌阿法狗的闻名，一股人工智能的浪潮正在席卷全世界，建议有空的时候研究研究。

史上最详尽“黑客马拉松”剖析【内附密探参与实录】

第二期

技术文章

1.SpringBoot的事务管理

之后每期都会有Spring boot的文章哟，本文简单介绍了Spring Boot在事务方面的配置和原理。

2.负载均衡原理的解析

本文由浅入深地介绍了各种负载均衡的方法和原理，看完本文你就会知道，使用Nginx反向代理做负载均衡只不过是负载均衡的方法之一，而且还不是在所有应用场景中最好的。

3.验证码安全那些事

本文介绍了各种各样的验证码的原理以及破解思路，所谓道高一尺魔高一丈，验证码从来就不是绝对安全的屏障，只是增加了羊毛党刷刷刷的成本而已。

推荐技术（或框架）：

Error-prone

谷歌出品的JAVA编译器，这个编译器和我们使用的javac编译器最大的不同是，它能额外地检查很多编程错误，甚至是bug，并用友好的格式打印出来

扩展阅读：

关于烂代码的那些事上篇 中篇 下篇
本文作者系新浪微博大数据技术专家和架构师，文章用诙谐幽默地风格，“喷”了作者在多年从业生涯中见到的各种各样的烂代码，并在后续篇章中给出了尽可能避免烂代码和重构代码的技巧。
这几篇文章在去年出来的时候一度非常火爆，引发很多人共鸣，在看完文章大笑之余，打开自己的项目，发现作者唠叨的东西出现在屏幕上，一定是一件非常有趣的事情。

第三期

技术文章

1.详解微信异步队列 MQ 2.0 的功能优化及拓展思路

由微信后台架构师廖文鑫分享的异步队列任务系统，对异步队列的架构设计有一定启发。

2.Leaf——来自美团点评的分布式ID生成系统

美团点评团队在业界是一支分享技术非常积极的团队（尤其前端），这篇文章介绍了常见分布式ID的生成方法和缺陷以及美团点评自己的一套生成ID的系统，具有参考价值。

3.如果有人问你数据库的原理，叫他看这篇文章 英文原文 中文译文

这是一篇英文文章，非常通俗易懂地讲清楚了关系型数据库底层的原理，对初学者有很大帮助，即使不是初学者，也能从中获得一些启发。有人已经把它翻译成了中文，降低了阅读成本，不过，毕竟翻译水平有限，有些地方可对照原文来看。

推荐技术（或框架）：

OkHttp

不用介绍了吧，这是JAVA平台上目前好评最多的http client，比apache之类的好用百倍，性能也有很大提高。

扩展阅读：

文章：我为什么不在乎人工智能
第一期曾经分享过有关深度学习的框架，因为从去年到今年人工智能的概念很火。而本文最近在网上比较火，因为它给火热的人工智能领域浇了一盆冷水。
人工智能究竟发展到什么程度了？人们想象的那种人工智能真的已经来了吗？人工智能的发展究竟瓶颈在哪里？阅读本文可以给你一些启示。

第四期

技术文章

1.Garbage Collectors - Serial vs. Parallel vs. CMS vs. G1 (and what’s new in Java 8)

都知道JVM有个垃圾回收器，但是你知道JVM有几个垃圾回收器吗？答案是4个！这篇文章简要介绍了每个垃圾回收器的特点，感兴趣就打开看看吧。

2.美团外卖订单中心的演进

技术文章高产赛母猪的美团点评团队给我们带来了美团外卖订单中心的架构演进，不妨观摩一下吧。

3.MySQL索引原理及慢查询优化

还是美团点评团队的文章，讲的很细致还有例子，好评~

推荐技术（或框架）：

java-design-patterns

本次推荐的既不是技术也不是框架，是一个github上的代码片段库。
该项目获得了18569个Star，在github的java版块排行榜里和Spring排在一起。
之所以这么有名，是因为这个代码片段库将JAVA语言所有的设计模式（90个）收集在了一起，给出了详细的结构图和示例代码。并随着时间推移，借助开源社区的力量增加，完善和改进。
如果你要学习设计模式，那么本项目绝对是第一选择。

扩展阅读：

文章：DevOps详解
什么是DevOps？DevOps为什么越来越火？DevOps的好处在哪儿？本文给你答案。

第五期

技术文章

1.How to select the first/least/max row per group in SQL

如题，MySQL经典书籍《高性能MySQL》作者Baron Schwartz在自己的博客上分享了一篇文章，讲述如何在group的时候，选取每组的特定数据(first/least/max)。实际上，这次我就遇到了同样的问题，所以才找到了这篇文章。

2.一文读懂 HTTP/2 特性

HTTP/2协议标准已经发表2年了，主流浏览器早就已经对其支持，即将发布的JDK9也将原生支持HTTP/2，那么新的HTTP协议究竟有什么好呢？会不会在未来几年爆炸式增长取代HTTP/1呢？

3.微服务架构下的分布式数据管理

微服务每个服务都有自己的数据库，如何保证数据一致性？用分布式事务来解决？那可不是那么容易的事儿。本文提出了另一种解决方案，具有一定的参考价值。

推荐技术（或框架）：

美团点评SQL优化工具SQLAdvisor开源
本次分享的是美团点评的开源工具，输入一段SQL，它就能有针对性地给出索引优化建议。

扩展阅读：

为何KPI毁了索尼，而OKR却成就了谷歌？
KPI说实话，是个古老的概念了，近几年硅谷衍生出来了新的概念——OKR，即以目标驱动的考核方式。KPI有什么缺陷导致索尼的前掌门人都承认它毁了索尼？OKR又有什么好处让谷歌如日中天？感兴趣就看一下咯。
