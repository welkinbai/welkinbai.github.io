今天打算把U盘里的博客生产环境做一份到我的笔记本电脑上，正好做的过程中可以从头再记录一遍这个博客的建设过程。

一、选择Hexo博客引擎

静态博客最近有多流行我就不用多说了吧，而静态博客里面最流行的引擎有两个：

  • jekyll
  • hexo

我的选择是hexo。原因如下：

  1. Hexo比jekyll生成简单，尤其是它的服务端命令，简单但是功能强大。可参照后文。
  2. Hexo和jekyll虽然都可以部署在github等第三方代码库上，但是Hexo只部署生成后的网页结果，而不是把全部的markdown文件、主题文件一并上传。这样比较保护作者的版权，否则别人直接一个Fork，你所有文章的原始文件就被拿到手了，主题文件也一并搞到手，分分种克隆一个相同的博客出来（连同你所有的文章），并换成他自己的名字。
  3. Hexo有大量好用的插件，参照后文
  4. Hexo可支持多个主题随心切换，只需要在配置文件里修改你要生效的主题名，就可以无缝换主题。jekyll的主题文件和其他文件混在一起，可能不是特别方便。

二、选择第三方代码托管库以及相应的page服务

  • 国外：github无疑
  • 国内：我本来选择的是gitcafe，无奈最近gitcafe与coding合并，于是变成了coding。

三、访问Hexo官网得到Hexo

说实话百度Hexo可没那么容易找到官网，通常你都会找到一个四轴飞行器的官网。
这是官网地址：https://hexo.io/zh-cn/

通过官网我们可以发现这是一个Node.js项目，要想使用需要先装Node.js。
这是Node.js官网地址：https://nodejs.org/en/
Node.js的windows版本是一个非常简单的安装包，安装之后你就拥有了Node.js，非常简单，零配置。
与此同时，还会为你安装Node.js的一个全局插件，即npm包管理器，可以从云端镜像服务器一键下载并安装Node.js的各种程序包。

现在你可以试试Hexo官网首页的命令了，win+R并输入cmd打开命令行窗口。然后试试。

npm install hexo-cli -g

四、使用Hexo搭建本地生产环境

在使用Hexo之前，你还需要安装以下git，也就是著名的代码管理工具，Hexo似乎在初始化的时候会使用git从它的代码库里克隆代码，因此没有git的话，后面使用Hexo会有少许问题。
git官网：https://git-scm.com/download/

安装好git，接下来就是Hexo大显身手的时刻了，你可以按照官网的步骤来做。也可以跟着我做：

这些文件和文件夹各有妙用：

  • node_modules：这个里面放着Hexo为你从Npm镜像源服务器下载的各种Hexo的插件，既有Hexo本身，也有Hexo的生成器和渲染器插件，以后你还可以自己通过Npm安装更多的插件，它们都会被放在这里。
  • scaffolds：这里放着3个markdown文件，分别对应草稿，页面，文章三种类型的页面模板，之后你使用Hexo new命令来新建一个markdown资源文件时，就可以指定使用哪个作为新建文件的模板。比如Hexo new post mynewpost，就会在资源文件夹Post分文件夹新建一个名为mynewpost.md的markdown文件。打开这个文件，写你的文章，之后Hexo博客引擎就能根据这个来直接生成静态的html页面并自动Push到你设置的服务器（或page静态服务的代码库）里。
  • source：资源文件夹，存放所有的文章的原始markdown文件。markdown是程序员世界最流行的撰写文章的工具，它是一套语言规范，可以使你像码代码一些快速码出带格式的文章。
  • themes：看英文名就知道，这是存放主题的地方，默认会为你下载一套还算不错的主题：landscape
  • .gitignore：这是git所使用的文件，里面会定义上传到page静态服务的git代码库时，忽略哪些文件或者文件夹，比如上文提到的node modules就完全没有必要上传到你自己的代码库。
  • _config.yml：这是你博客的全局配置，博客名，博客要使用的主题等等，Hexo渲染你的博客时，会根据这里面的配置进行一些调整。你的代码库也配置在这里。
  • package.json：这是Node.js规范里指定的，一个Node.js的项目一定会有这样的配置文件来告诉Node.js这个项目叫什么，版本号是什么，有哪些依赖。

七、部署到因特网

现在博客只能在你自己本地看到，如何将它部署到网络上让所有人都看到呢？
一种方法是你购买服务器或者虚拟服务器，然后部署到上面。
然而大多数程序员都选择第二种方法：选择github等第三方代码库提供的免费的Page静态服务来部署博客。
只需要将相关html上传到代码库中，github就会为你自动部署，你甚至还可以弄个域名，替换掉默认的<用户名>.github.io。

拿Github举例。
github官网：https://github.com

九、创建一个可携带的生产环境

本文的重点来了，这是我自创的。
如果你细心的话就会发现，Hexo写博客每次都只能打开你本地的某个文件夹，然后Hexo new，然后写博客，然后hexo g，hexo d。

问题来了，如果你的电脑不在身边呢？如果你临时想用另一台电脑写博客呢?
答案是把本地的这个文件夹装进U盘啊。这样不就可以走哪儿写哪儿了吗？

没那么简单，有了这个想法，付出实践还是费了一番功夫的。

  1. 首先需要解决的就是Hexo的依赖问题。Hexo依赖于Node.js和Git。但是不是每个电脑都装好了它们的。
    因此，我需要想办法把它们变成便携版。
    Git官网有便携版可以下载，下载下来后，直接就可以放在U盘里使用，非常贴心。
    而Node.js，尽管没有便携版，但是我发现它是绿色的，整个拷贝C盘中Node.js的文件夹到U盘，便可作为便携版使用。

  2. 现在U盘环境搭好了，但是每次写文章输入一大堆命令真是不方便。别的不说，光指定环境变量就够麻烦的了，于是我又写了windows脚本。

十、写在最后

写博客文章实在是太费时间了，所以我可能写的不是特别详细，如果你有什么问题，大可在评论里询问我。
