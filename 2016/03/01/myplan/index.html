这世界变化太快,计划都是童话

2016年3月1日 | 置顶, 计划

新的人生大计

新的计划………………

由于换了公司，业余时间变少了，之前的计划作废，调整一个新的计划：

周一到周五，技术栈学习。
周末，LC项目或者其他github项目开发

技术栈安排：
2016.10~12 高性能Mysql读书笔记（已完成）
2016.12~2017.01 mybatis深入实践（已完成）
2017.01~2017.03 设计模式/effective java
待定 JDK基础和源代码解析（注解反射、集合类、IO、NIO、日期类、并发包、其他工具包、1.8新特性专题）
待定 Spring第二遍~~

===================无耻地分割线===========================

除了spring和js过了一遍以外之前计划的都没能完成，实在太水了。我决定制定一个更加详细有弹性的计划。
废话不多说，开始：

周一 技术栈学习
周二 CodingAllInAll
周三 技术栈学习
周四 github项目开发
周五 娱乐时间
周六早上 CodingAllInAll
周六下午 健身
周六晚上 弹性时间(周一到周四有因为加班没能完成的，就这个时候补上)
周日 LC项目

技术栈近期安排：
2016.08~09 高性能Mysql读书笔记
2016.09~10 mybatis深入实践
2016.10~11 JDK基础（注解反射、集合类、IO、NIO、日期类、并发包、其他工具包、1.8新特性专题）
2016.11~12 Spring第二遍

博客建好后一直都非常忙，但是建这个博客的目的主要是为了能记录自己学习的进度，鞭策自己。
但是没想到这么蹉跎， 还是为自己制定个计划吧。

## Spring 第一阶段学习(2016-03~2016-07)
今天下班前倒腾半天都没把一个有关Spring事务的功能搞定，妈蛋，非常不爽。
所以目前将Spring的学习提前到第一优先级。它已经开始影响我的工作了。
而且你要知道，Spring使用了很多一眼看不穿的机制，一旦出错，错误往往很诡异，让人摸不着头脑，浪费大量时间。

## JAVA JDK 1.8 源码学习(2016-04~2016-06)
这个计划是我早就想执行的，现在IDE已经装好，抽时间一定要做这件事。
我想，作为一个JAVA工程师，JAVA官方的代码无论如何也要有些了解吧。
如果想成为一个大师，那么JDK是必须彻底地熟知才行。
况且我现在刚开始学习，我想最好找人们公认的好代码来模仿，那么没有比JDK更合适的了。
当然，还有Spring的源码，等这个计划完成，我想就可以进一步来学习Spring的源码。

## JS 语法学习（2016-04~2016-06）
JS语言可以撬开前端世界的大门，我现在非常想进入前端的世界。
从我目前的经历来看，我个人更加倾向于自由，易于测试的前端世界。
而且我平时自己爱折腾一些小工具，利用前端语言可以很轻易地部署这些小工具到页面上，但是后端就不行了，尤其是JAVA，非常地重，很难去快速地部署一个小型应用。
现在又有了node.js，让我多了一个学习JS的理由。在我心目中，node.js是非常有活力的后端语言，以后只要JS在手，就能前后端通吃联调，这是非常好的事情。毕竟代码开发涉及的人越多，质量越难保证。

## 更多计划？
还有更多计划，我想先不要想太多，完成已经提出的已经很不容易了，毕竟提出太多计划不是个好主意。
等到前两个计划开始进行一半的时候，我会更新这篇文章。
